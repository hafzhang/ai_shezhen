## 三、成本优化建议

### 3.1 Prompt 优化

1. **精简 System Prompt**
   - 当前: ~500 tokens
   - 优化后: ~300 tokens
   - 节省: ~0.0008 元/次

2. **精选 Few-shot 示例**
   - 当前: ~1500 tokens (5个完整示例)
   - 优化后: ~900 tokens (3个精选示例)
   - 节省: ~0.0024 元/次

3. **动态示例选择**
   - 根据用户特征选择最相关的 2-3 个示例
   - 可节省 ~50% 的示例 token

### 3.2 缓存策略

1. **相似病例缓存**
   - 对相同舌象特征的诊断结果缓存 24 小时
   - 预期缓存命中率: 30-40%
   - 实际调用量减少: 30-40%

2. **Token 缓存**
   - System Prompt 和 Few-shot 示例可作为静态内容
   - 利用平台的缓存机制（如果支持）

### 3.3 模型选择策略

1. **分级模型使用**
   - 简单病例: ERNIE-Lite (¥0.003/千tokens)
   - 复杂病例: ERNIE-Speed (¥0.004/千tokens)
   - 专家会诊: ERNIE-4.0 (¥0.012/千tokens)

2. **成本节省估算**
   - 60% 简单病例使用 Lite: 节省 25%
   - 综合成本可降低 15-20%

### 3.4 兜底方案

1. **本地规则库**
   - 覆盖常见病例 (80%+)
   - 仅 20% 疑难病例调用 LLM
   - 可节省 80% 的 API 成本

2. **降级策略**
   - API 超时或连续失败时自动切换
   - 预设 200+ 条规则库

## 八、成本计算示例代码

```python
def calculate_diagnosis_cost(
    system_prompt_tokens: int = 500,
    few_shot_tokens: int = 1500,
    user_input_tokens: int = 300,
    expected_output_tokens: int = 800,
    input_price: float = 0.004,  # 元/千tokens
    output_price: float = 0.004  # 元/千tokens
) -> dict:
    """
    计算单次诊断成本

    Returns:
        dict: 包含各项 token 数量和成本的字典
    """
    total_input = system_prompt_tokens + few_shot_tokens + user_input_tokens

    input_cost = (total_input / 1000) * input_price
    output_cost = (expected_output_tokens / 1000) * output_price
    total_cost = input_cost + output_cost

    return {
        "input_tokens": total_input,
        "output_tokens": expected_output_tokens,
        "total_tokens": total_input + expected_output_tokens,
        "input_cost": input_cost,
        "output_cost": output_cost,
        "total_cost": total_cost
    }

# 使用示例
cost = calculate_diagnosis_cost()
print(f"单次诊断成本: ¥{cost['total_cost']:.4f}")
```
