## Codebase Patterns

### Database & ORM
- Use `sqlalchemy >= 2.0` with `future=True` for modern ORM style
- Use `UUID(as_uuid=True)` with `server_default=func.gen_random_uuid()` for primary keys
- Use `JSONB` field type for flexible schema-less data storage
- Use `has_key()` for checking JSONB key existence in SQLAlchemy queries
- Use `field.astext` for text extraction from JSONB fields
- Use `func.date()` for grouping by date in time-series queries
- Foreign key with `ondelete="CASCADE"` for automatic cleanup of related records
- Foreign key with `ondelete="SET NULL"` for preserving records when parent is deleted

### Authentication & Authorization
- Use `get_current_user()` dependency for required authentication
- Use `get_optional_user()` dependency for optional authentication (anonymous support)
- Use `get_current_active_user()` to additionally check `deleted_at is NULL`
- JWT access token expires in 30 minutes
- JWT refresh token expires in 7 days
- Password hashing uses bcrypt via `passlib.context.CryptContext`
- JWT secrets must be changed in production (`openssl rand -hex 32`)

### API Design (v2)
- All v2 endpoints extend `APIResponse` base model for unified response format
- Response models include `success: bool` field
- Use `HTTPException` with proper status codes (400, 401, 403, 404, 500)
- Pagination uses `page` (1-indexed) and `page_size` parameters
- Date filters use ISO 8601 format (`2026-01-01T00:00:00Z`)
- Ownership verification: compare `resource.user_id == current_user.id`
- Anonymous users supported by storing `user_id = NULL`

### Pydantic Models
- Use `Field()` for all model fields with descriptions
- Use `@field_validator` for custom validation logic
- Use `model_config = {"from_attributes": True}` for ORM mode
- Use `ge=`/`le=` for numeric constraints
- Use `max_length=` for string constraints

### Configuration
- Use `pydantic-settings` (`BaseSettings`) for environment variables
- Add `MEDIA_ROOT` to settings for file storage paths
- Use `model_config = SettingsConfigDict(extra="ignore")` to ignore unknown env vars
- Environment file: `.env` in project root

### Model References
- Create `set_model_references()` function in endpoint modules
- Call `set_model_references()` in `initialize_models()` after model loading
- Use fallback imports to get models from v1/endpoints if not set locally

### File Storage
- Calculate SHA-256 hash for image deduplication
- Store images in `MEDIA_ROOT/tongue_images/{hash}.png`
- Use `file_hash` as unique key for database deduplication

### Error Handling
- Use `try/except` with specific exception types
- Log errors with `logger.error()` including `exc_info=True`
- Use `db.rollback()` on database errors before re-raising
- Return structured error responses with `detail` field
