# 百度AI舌诊解决方案完整文档

## 一、方案概述

本文档详细介绍如何使用百度AI模型构建智能舌诊分析系统。舌诊是中医诊断的重要方法，通过观察舌象（舌质、舌苔、舌形）来判断人体健康状况。

## 二、百度AI模型选择指南

### 2.1 核心推荐方案

| 需求场景 | 推荐模型/服务 | 获取方式 | 说明 |
|---------|-------------|---------|------|
| 舌体图像分割 | PaddleSeg | paddlepaddle.org.cn | 图像分割套件，精确提取舌体区域 |
| 舌象分类识别 | PaddleClas | paddlepaddle.org.cn | 图像分类套件，支持舌色、舌苔分类 |
| 定制化训练 | EasyDL | cloud.baidu.com | 零代码训练舌象分类模型 |
| 智能诊断建议 | 文心一言 (ERNIE) | qianfan.cloud.baidu.com | 大语言模型，生成健康分析报告 |
| OCR文字识别 | PaddleOCR | github.com/PaddlePaddle/PaddleOCR | 识别舌诊报告中的文字 |

### 2.2 方案一：云端API方案（快速部署）

**适合场景**：快速原型开发、小型应用

**使用服务**：
- 百度智能云图像识别API
- 文心一言对话API

**优势**：
- 无需本地部署模型
- 开发速度快
- 自动更新维护

**限制**：
- 需要网络连接
- API调用有费用
- 数据需上传云端

### 2.3 方案二：本地部署方案（数据隐私）

**适合场景**：医疗机构、注重隐私保护

**使用框架**：
- PaddlePaddle深度学习框架
- 本地部署开源模型

**优势**：
- 数据完全本地化
- 无网络依赖
- 可自由定制

**限制**：
- 需要一定的开发能力
- 需要硬件支持

### 2.4 方案三：混合方案（推荐）

**架构**：
```
本地处理：图像分析、舌体分割、特征提取
    ↓
云端增强：大模型生成诊断建议
```

**优势**：
- 平衡性能与智能
- 灵活可控
- 成本相对较低

## 四、舌诊技术方案详解

### 4.1 舌体分割技术

**目的**：从照片中精确提取舌体区域，排除嘴唇、牙齿等干扰。

**技术方案**：

| 方案 | 精度 | 速度 | 难度 |
|-----|-----|-----|-----|
| 颜色阈值法 | 中 | 快 | 低 |
| 传统机器学习 | 中 | 中 | 中 |
| PaddleSeg分割 | 高 | 中 | 中 |
| 自定义训练 | 最高 | 慢 | 高 |

**推荐模型**（PaddleSeg）：
- DeepLabV3+：精度高，适合医学图像
- UNet：轻量级，适合实时应用
- HRNet：兼顾精度和速度

### 4.2 舌象特征分析

**分析维度**：

#### (1) 舌质颜色
```
淡白舌 ───→ 气血不足、阳虚
淡红舌 ───→ 正常健康
红舌 ────→ 热证（实热/虚热）
紫舌 ────→ 血瘀、寒凝
青舌 ────→ 寒盛、瘀血
```

#### (2) 舌苔特征
```
薄白苔 ───→ 正常表证
白厚苔 ───→ 寒湿、食积
黄苔 ────→ 里热证
黄厚苔 ───→ 实热、湿热
少苔/无苔 → 阴虚、胃气不足
灰黑苔 ───→ 里热或里寒重证
```

#### (3) 舌形形态
```
正常 ────→ 气血调和
胖大 ────→ 阳虚水湿、痰湿
瘦薄 ────→ 阴虚、气血不足
齿痕 ────→ 脾虚湿盛
裂纹 ────→ 阴血不足、热盛伤阴
点刺 ────→ 热毒内盛
```

### 4.3 智能诊断建议

**文心一言应用**：

输入示例：
```
舌象分析结果：
- 舌质颜色：红舌
- 舌苔：薄黄苔
- 舌形：正常
- 其他：舌尖有红点

请根据中医舌诊理论，分析可能的健康状况并给出调理建议。
```

输出示例：
```
根据舌象分析：
1. 舌质偏红、苔薄黄，提示体内有热象
2. 舌尖红点多与心火有关
3. 综合判断：心火亢盛证

调理建议：
- 饮食：清淡为主，避免辛辣
- 茶饮：莲子心茶、菊花茶
- 作息：保证充足睡眠
- 情绪：保持心情舒畅

注意：以上仅供参考，具体请咨询专业中医师
```

## 五、API集成指南

### 5.1 百度智能云通用API

**认证方式**：OAuth 2.0

**获取Access Token**：
```
POST https://aip.baidubce.com/oauth/2.0/token

参数：

```

```json
{
  "messages": [

      "role": "user",

  ],
  "temperature": 0.7,
```

```
                         ↓

│ - 图像预处理      │ - 舌体分割       │ - 文心一言   │
```

### 6.2 数据流

```
1. 用户上传舌象照片
        ↓
2. 图像质量检测（清晰度、光照、角度）
        ↓
3. 舌体区域分割
        ↓
4. 特征提取（颜色、纹理、形状）
        ↓
5. 舌象分类识别
        ↓
6. 结合中医知识库分析
        ↓
7. 文心一言生成诊断建议
        ↓
8. 输出完整报告
```

## 七、数据准备与标注

### 7.1 数据采集要求

| 项目 | 要求 |
|-----|------|
| 拍摄环境 | 自然光或标准光源 |
| 拍摄角度 | 正面拍摄，舌体自然伸出 |
| 图片分辨率 | 建议1080p及以上 |
| 图片格式 | JPG/PNG |
| 背景要求 | 纯色背景，避免干扰 |

### 7.2 数据标注规范

**舌质颜色标注**：
- 淡白舌
- 淡红舌
- 红舌（分淡红、鲜红、深红）
- 紫舌
- 青舌

**舌苔标注**：
- 薄白苔
- 白厚苔
- 薄黄苔
- 黄厚苔
- 少苔/无苔
- 灰黑苔
- 腻苔

**舌形标注**：
- 正常
- 胖大
- 瘦薄
- 齿痕
- 裂纹
- 点刺

### 7.3 数据集建议

**最小数据集**：每个类别至少100张图片

**推荐数据集**：
- 训练集：70%
- 验证集：15%
- 测试集：15%

**数据增强**：
- 亮度调整
- 对比度调整
- 轻微旋转
- 水平翻转（慎用，舌象有左右之分）

- 腾讯云CVM

```dockerfile
WORKDIR /app
```

## 十一、常见问题

### Q1: 哪个百度模型最适合舌诊？
A: 推荐混合方案：PaddleSeg（分割）+ PaddleClas（分类）+ 文心一言（分析）

### Q2: 需要多少数据才能训练模型？
A: 最少每个类别100张，建议500-1000张以获得更好效果

### Q3: API调用费用如何？
A: 文心一言按Token计费，约0.008元/千Token，有免费额度

### Q4: 如何提高识别准确率？
A:
1. 增加训练数据量
2. 规范数据采集标准
3. 使用数据增强
4. 模型融合

### Q5: 能否离线使用？
A: 使用本地部署方案可以完全离线运行
