## Codebase Patterns
- The dataset uses COCO format with train/val/test splits
- Image files have irregular filenames (with parentheses like 'F (1).jpg')
- Categories are defined in classes.txt files with Chinese names
- Category IDs start from 0 (jiankangshe) to 20 (xinfeitu)
- The annotation file may reference category_id 21 which is not defined in categories
- Mask files are stored as PNG with binary values (0=background, 255=tongue)
- COCO annotations contain polygon coordinates that need conversion using cv2.fillPoly
- Multiple annotations per image are merged using bitwise OR operation
- Class mapping: 21 original classes map to 6 dimensions (tongue_color:4, coating_color:4, tongue_shape:3, coating_quality:3, features:3, health:1)
- Syndrome classes (13-20) are excluded from classification training and reserved for LLM few-shot prompts
- Label files use TAB-separated format: filename<TAB>label1,label2,...,label19 (one-hot vector)
- Class weights formula: weight_i = total / (num_classes × count_i) - higher weights for minority classes
- For stratified sampling with no minority/majority classes, use fallback to random sampling from all classes
- torch.utils.data.Sampler requires __iter__ and __len__ methods; BatchSampler yields lists of indices
- PaddlePaddle CrossEntropyLoss uses functional API with axis=1 for (N,C,H,W) input shape
- PaddlePaddle 3.x uses nn.CrossEntropyLoss() but F.cross_entropy() for multi-dimensional inputs
- For one_hot encoding, handle ignore_index before calling F.one_hot() to avoid errors
- Albumentations 2.0+ uses new API: GaussNoise uses std_range instead of var_limit, CoarseDropout uses max_size instead of max_height/max_width
- Classification dataset uses labels.txt with 19-dimensional one-hot vectors, images located in seg_v1/train/images
- PaddlePaddle 3.x PTQ API: use PTQ and PTQConfig for post-training quantization; QAT API may have compatibility issues
- FP16 conversion: Convert state dict parameters to float16 numpy arrays before saving with paddle.save()
- Windows console needs UTF-8 encoding wrapper for Unicode characters in Chinese filenames
- Hard example mining (OHEM): Use deque with maxlen for sliding window loss smoothing; track sample losses across epochs
- Progressive training: Freeze detail/context modules separately for phased training; lower LR for detail branch phase
- HardExampleDataset: Augment hard examples more aggressively (3x factor) than base training samples
- scipy.ndimage operations: Handle 3D (images) vs 2D (masks) arrays differently for rotation/elastic transform
- PP-HGNetV2-B4 architecture: 3 parallel LightConvBNReLU branches per HG-Block, feature_dim=864
- Multi-head classification: Each head can have different loss weights, num_classes, and multi-label settings
- Multi-head model structure: Backbone (864-dim) -> 6 heads with configurable hidden layers and dropout
- PaddleClas setup: Use LayerDict for named heads, register_buffer for loss weights as tensors
