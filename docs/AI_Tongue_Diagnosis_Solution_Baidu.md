# 基于百度飞桨 (PaddlePaddle) 的 AI 舌诊技术方案

## 1. 方案概述
使用百度的技术栈构建 AI 舌诊系统，目前**没有单一的“舌诊模型”**可以直接完成所有任务。通用的工业级解决方案是通过组合百度 **PaddlePaddle (飞桨)** 开源算法库中的多个专用模型，构建一个“**检测 -> 分割 -> 识别 -> 诊断**”的完整流水线。

## 2. 核心模型选型推荐

针对舌诊流程的各个阶段，推荐使用以下百度飞桨官方模型：

| 步骤 | 任务描述 | 推荐模型 (Paddle系) | 推荐理由 |
| :--- | :--- | :--- | :--- |
| **Step 1** | **舌体检测** (Object Detection)<br>从面部图像中定位舌头区域。 | **PP-PicoDet** (移动端)<br>**PP-YOLOE** (服务端) | **PP-PicoDet** 是目前移动端最快的检测模型，适合手机端实时检测；服务端追求高精度推荐 PP-YOLOE。 |
| **Step 2** | **舌体分割** (Semantic Segmentation)<br>将舌头从背景（嘴唇、牙齿）中精确抠出。 | **PP-LiteSeg**<br>或 **DeepLabV3+** | **PP-LiteSeg** 是百度自研的 SOTA 实时分割模型，速度极快且边缘分割精准，对分析舌苔边缘至关重要。 |
| **Step 3** | **特征识别** (Classification)<br>识别舌色、苔色、齿痕、点刺等特征。 | **PP-LCNet**<br>或 **PPLCNetV2** | **PP-LCNet** 是专为 CPU 优化的轻量级骨干网络，识别速度极快，适合部署在各类终端。 |
| **Step 4** | **辨证诊断** (Reasoning)<br>基于特征输出中医诊断与建议。 | **文心一言 (ERNIE Bot)** | 将识别出的结构化特征（如“舌红、苔黄腻”）作为 Prompt 发送给文心大模型，生成专业的调理建议。 |

## 3. 技术架构与处理流程

一个标准的 AI 舌诊处理流程如下：

1.  **图像采集与预处理**：
    *   用户拍摄舌像。
    *   **白平衡/色彩校正**（关键）：使用 OpenCV 或传统算法校正光照影响，确保颜色真实。
2.  **舌体检测 (Detection)**：
    *   输入：原始图片。
    *   模型：PP-PicoDet。
    *   输出：舌头的矩形框坐标 (ROI)。
3.  **舌体分割 (Segmentation)**：
    *   输入：裁剪后的舌头 ROI 图片。
    *   模型：PP-LiteSeg。
    *   输出：舌头掩膜 (Mask)，去除嘴唇、牙齿等干扰背景。
4.  **特征提取 (Feature Extraction)**：
    *   输入：经过掩膜处理的纯净舌头图像。
    *   模型：PP-LCNet (多个分类头)。
    *   输出：
        *   舌质：淡红、红、绛、紫...
        *   舌苔颜色：白、黄、灰黑...
        *   舌苔厚度：薄、厚、腻、腐...
        *   形态特征：齿痕、点刺、裂纹...
5.  **智能诊断 (Diagnosis)**：
    *   输入：上述特征的文本描述。
    *   模型：文心一言 API。
    *   输出：中医证型（如“湿热内蕴”）及健康调理建议。

## 4. 开发与落地路径

### 方案 A：EasyDL 零代码开发平台（适合快速验证）
*   **适用人群**：无深度学习算法背景的开发者、初创团队。
*   **流程**：
    1.  在百度 EasyDL 平台上传标注好的舌头数据集。
    2.  在界面上选择“物体检测”和“图像分割”任务。
    3.  平台自动训练并发布 API 接口。
*   **优点**：上手快，无需编写底层模型代码。

### 方案 B：PaddlePaddle 深度定制开发（适合企业级商用）
*   **适用人群**：算法工程师、需要私有化部署或高性能场景。
*   **流程**：
    1.  **PaddleDetection**：训练检测模型。
    2.  **PaddleSeg**：训练分割模型。
    3.  **PaddleClas**：训练多标签分类模型。
    4.  **PaddleLite / FastDeploy**：将模型部署到安卓/iOS或服务器端。
*   **优点**：性能极致，可完全掌控模型细节，成本可控。

## 5. 关键挑战与建议

1.  **光照影响**：舌诊对颜色非常敏感。建议在 App 端引导用户在自然光下拍摄，并在算法侧引入**传统色彩校正算法**（如 Gray World Assumption）作为预处理。
2.  **数据积累**：模型的准确率高度依赖训练数据。建议初期通过开源医学数据集或合作医院收集高质量标注数据（至少数百张起步）。
3.  **边缘设备性能**：如果在手机端离线推理，务必选择 **PP-PicoDet** 和 **PP-LCNet** 等轻量化模型，以保证用户体验流畅。
