### 2.5 数据预处理流水线（优化版）

**针对类别不平衡的特殊处理**：

```
┌──────────────────────────────────────────────────────────┐
│ 阶段0：类别不平衡分析与应对策略（新增）                      │
│ - 统计各类别分布，识别多数类和少数类                         │
│ - 计算类别权重：weight_i = total / (num_classes × count_i)  │
│ - 确定过采样/欠采样阈值：<100样本过采样，>1000样本欠采样     │
│ - 策略选择：Focal Loss + 分层采样 + 困难样本挖掘            │
└──────────────────────────────────────────────────────────┘
   │
   ▼

**技术实现细节补充**：
- **Polygon填充库**: 使用 `pycocotools.mask` 或 `cv2.fillPoly` 将COCO polygon转换为二值mask
- **颜色空间**: 推荐使用 **Lab颜色空间** 进行增强，更符合人眼感知（L=亮度, a=绿-红, b=蓝-黄）
- **增强库**: 推荐使用 **Albumentations**（医学图像专用增强库）
- **医学图像专用增强**: 染色归一化（Stain Normalization）减少不同设备拍摄差异

```

│ - 图像完整性检查（损坏/空文件剔除）                           │
│ - 标注一致性校验（COCO polygon vs YOLO bbox IoU > 0.9）    │

### 2.6 数据集利用率优化策略

| 策略 | 说明 | 预期增益 | 针对问题 |
|------|------|----------|----------|
| 预训练权重迁移 | ImageNet22k → PaddleClas | 准确率+5~8% | 小样本学习 |
| **分层采样** | 每batch保证各类别均衡 | **少数类召回+30%** | **类别不平衡** |
| **Focal Loss** | 动态调整难易样本权重 | **少数类精度+15%** | **类别不平衡** |
| **类别重加权** | loss_weight = 1/√(count) | 少数类关注+20% | 类别不平衡 |
| 半监督学习 | 利用未标注图像 | 有效样本+8% | 数据量不足 |
| 主动学习 | 困难样本优先标注 | 训练效率+30% | 标注成本 |
| 知识蒸馏 | 大模型→小模型 | 轻量化部署 | 部署优化 |
| Mixup/CutMix | 样本混合增强 | 鲁棒性+10% | 过拟合预防 |

### 2.7 类别不平衡专项应对方案（新增）

**三层应对策略**：

```
┌─────────────────────────────────────────────────────────────┐
│ 第一层：数据层面                                          │
├─────────────────────────────────────────────────────────────┤
│ • 过采样：少数类复制+轻微变换                               │
│ • SMOTE合成：生成中间样本                                  │
│ • 欠采样：多数类随机丢弃（保持80%）                          │
│ • 目标：各类别比例控制到 < 10:1                            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第二层：算法层面                                          │
├─────────────────────────────────────────────────────────────┤
│ • Focal Loss：α=0.25, γ=2（聚焦难样本）                    │
│ • 类别加权Loss：weight = 1/√(count)                        │
│ • 分层采样：每batch包含多数类+少数类各50%                   │
│ • 难例挖掘：loss Top 10%样本重采样                           │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第三层：评估层面                                          │
├─────────────────────────────────────────────────────────────┤
│ • 使用宏平均F1（而非准确率）                                │
│ • 分别关注多数类和少数类的指标                              │
│ • 混淆矩阵分析找出易混淆类别对                             │
│ • 少数类单独评估（不与多数类混合比较）                      │
└─────────────────────────────────────────────────────────────┘
```

**具体实施代码示例**：

```python
# 类别权重计算（针对舌诊数据集）
class_weights = {
    0: 1.0,    # botaishe (2578 samples)
    1: 3.5,    # hongshe (1279 samples) ← 加权
    2: 6.2,    # zishe (209 samples)   ← 高权重
    3: 2.0,    # pangdashe (622 samples)
    4: 2.5,    # shoushe
    5: 3.0,    # hongdianshe
    6: 2.8,    # liewenshe
    7: 2.6,    # chihenshe
    8: 3.0,    # baitaishe
    9: 5.0,    # huangtaishe
    10: 8.0,   # heitaishe          ← 极少类最高权重
    # ... 证型类不参与分类训练
}

# 分层采样器
from torch.utils.data import WeightedRandomSampler

sample_weights = []
for idx, (data, target) in enumerate(dataset):
    # target是多标签，取权重最大的类别
    class_ids = target.nonzero().squeeze()
    if len(class_ids) > 0:
        max_weight = max([class_weights[cid.item()] for cid in class_ids])
        sample_weights.append(max_weight)
    else:
        sample_weights.append(1.0)

sampler = WeightedRandomSampler(sample_weights, num_samples=len(sample_weights))
```

### 3.2 Stage 1：舌体分割模型 (PaddleSeg)

| 配置项 | 方案（基于5404样本 + 类别不平衡优化） |

**训练目标**（基于数据规模调整）：
- mIoU > 0.92
- Dice > 0.95
- 推理速度 > 30 FPS (CPU), >80 FPS (GPU)
- 模型大小 < 10MB

**训练技巧**（优化）：
1. **早停机制**：val mIoU连续10个epoch不提升则停止，防止过拟合
2. **渐进式训练**：前30epoch训练细节分支，后50epoch联合训练
3. **难例挖掘**：每epoch选取loss最高的10%样本加入下一epoch
4. **边界优化**：使用BoundaryLoss强化舌体边缘分割
5. **学习率预热**：仅2epoch warmup，快速进入正常学习
6. **模型EMA**：使用指数移动平均保存模型，提升泛化能力

**防止过拟合策略**（小数据集关键）：
- Dropout(0.1) 在分割头中
- 权重衰减 weight_decay=5e-4
- 早停 patience=10
- 数据增强强度随epoch增加（课程学习）

| 方案 | 说明 | 适用场景 | 预期增益 |
|------|------|----------|----------|
| **Vote Ensemble** | 多模型投票 | 不同架构模型互补 | mAP +2~3% |
| **Weighted Fusion** | 加权融合 | 精度可量化的模型 | mAP +3~5% |
| **Stacking** | 二层学习器 | 大量训练数据 | mAP +5~8% |
| **Cascade** | 级联推理 | 性能敏感场景 | 速度+30% |

**具体实施方案**：
```python
# Vote Ensemble示例
predictions = []
for model in [model_seg, model_clas_v1, model_clas_v2]:
    pred = model.predict(image)
    predictions.append(pred)

# 多数投票
final_pred = np.mean(predictions, axis=0)
final_pred = (final_pred > 0.5).astype(int)

# 加权融合（根据验证集精度）
weights = [0.4, 0.35, 0.25]  # 各模型验证集权重
final_pred = np.average(predictions, axis=0, weights=weights)
```

    ↓                       ↓

                              ↓

### 附录J：舌诊数据集特殊处理指南（新增）

#### J1. 不规则文件名处理

**问题描述**：数据集图像命名不规则，如 "A (1000).jpg", "C (354).jpg" 等

**解决方案**：
```python
import os
import glob
from PIL import Image

class TongueDataset(torch.utils.data.Dataset):
    def __init__(self, root_dir, transform=None):
        self.root_dir = root_dir
        self.image_dir = os.path.join(root_dir, 'images')
        self.label_dir = os.path.join(root_dir, 'labels')

        # 获取所有图像文件（处理不规则命名）
        self.images = glob.glob(os.path.join(self.image_dir, '*.jpg'))
        self.images += glob.glob(os.path.join(self.image_dir, '*.JPG'))
        self.images += glob.glob(os.path.join(self.image_dir, '*.png'))

        # 建立图像到标签的映射
        self.name_to_label = {}
        for label_file in glob.glob(os.path.join(self.label_dir, '*.txt')):
            img_name = os.path.basename(label_file).replace('.txt', '.jpg')
            self.name_to_label[img_name] = label_file

    def __getitem__(self, idx):
        img_path = self.images[idx]
        img_name = os.path.basename(img_path)

        # 加载图像
        image = Image.open(img_path).convert('RGB')

        # 加载对应的YOLO标签
        label_path = self.name_to_label.get(img_name)
        if label_path and os.path.exists(label_path):
            boxes, labels = self.load_yolo_label(label_path)
        else:
            # 如果没有对应的YOLO标签，尝试使用COCO标注
            boxes, labels = self.load_coco_label(img_name)

        if self.transform:
            image = self.transform(image)

        return image, boxes, labels
```

#### J2. 类别不平衡数据处理

**分层采样器实现**：
```python
from torch.utils.data import Sampler
import numpy as np

class StratifiedSampler(Sampler):
    def __init__(self, dataset, batch_size, num_samples=None):
        self.dataset = dataset
        self.batch_size = batch_size

        # 统计每个样本的类别权重
        self.weights = self._calculate_weights()
        self.num_samples = num_samples or len(self.weights)

    def _calculate_weights(self):
        weights = []
        for idx in range(len(self.dataset)):
            _, _, labels = self.dataset[idx]
            if len(labels) > 0:
                # 取最小权重（对应最多类别）
                min_weight = min([CLASS_WEIGHT.get(l, 1.0) for l in labels])
                weights.append(min_weight)
            else:
                weights.append(1.0)
        return torch.DoubleTensor(weights)

    def __iter__(self):
        return iter(torch.utils.data.WeightedRandomSampler(
            self.weights, self.num_samples, replacement=False
        ))

    def __len__(self):
        return self.num_samples // self.batch_size
```

def validate_dataset(root_dir):
    """验证数据集完整性"""
    issues = []

    missing_labels = []
    for img_path in glob.glob(os.path.join(root_dir, 'images', '*.jpg')):

        if not os.path.exists(label_path):
            missing_labels.append(img_name)

    if missing_labels:
        issues.append(f"缺少标签文件: {len(missing_labels)}个")

    # 2. 检查损坏图像
    from PIL import ImageFile
    ImageFile.LOAD_TRUNCATED_IMAGES = True
    corrupted = []
    for img_path in glob.glob(os.path.join(root_dir, 'images', '*.jpg')):
        try:
            with Image.open(img_path) as img:
                img.verify()
        except Exception as e:
            corrupted.append(os.path.basename(img_path))

    if corrupted:
        issues.append(f"损坏图像: {len(corrupted)}个")

    return issues
