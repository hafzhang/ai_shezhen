## Codebase Patterns
- The dataset uses COCO format with train/val/test splits
- Image files have irregular filenames (with parentheses like 'F (1).jpg')
- Categories are defined in classes.txt files with Chinese names
- Category IDs start from 0 (jiankangshe) to 20 (xinfeitu)
- The annotation file may reference category_id 21 which is not defined in categories
- Mask files are stored as PNG with binary values (0=background, 255=tongue)
- COCO annotations contain polygon coordinates that need conversion using cv2.fillPoly
- Multiple annotations per image are merged using bitwise OR operation
- Class mapping: 21 original classes map to 6 dimensions (tongue_color:4, coating_color:4, tongue_shape:3, coating_quality:3, features:3, health:1)
- Syndrome classes (13-20) are excluded from classification training and reserved for LLM few-shot prompts
- Label files use TAB-separated format: filename<TAB>label1,label2,...,label19 (one-hot vector)
- Class weights formula: weight_i = total / (num_classes × count_i) - higher weights for minority classes
- For stratified sampling with no minority/majority classes, use fallback to random sampling from all classes
- torch.utils.data.Sampler requires __iter__ and __len__ methods; BatchSampler yields lists of indices
