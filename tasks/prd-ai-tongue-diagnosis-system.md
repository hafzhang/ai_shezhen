## 一、Introduction/Overview

### 1.1 项目简介

AI舌诊智能诊断系统是一个基于计算机视觉和大型语言模型的中医舌诊辅助诊断系统。系统通过分析用户上传的舌部图像，自动提取舌象特征（舌色、苔色、舌形、苔质、特殊特征等），并结合用户基本信息进行中医辨证分析，提供个性化的健康建议。

### 1.2 核心价值

- **辅助诊断**: 为中医师提供客观的舌象特征分析，减少主观判断偏差
- **健康管理**: 帮助普通用户了解自身舌象特征，获取中医养生建议
- **教育价值**: 为中医学习者提供标准化的舌象识别训练工具

## 二、Goals

### 2.1 核心目标

1. **高精度舌体分割**: 实现mIoU > 0.92的舌体分割精度
2. **准确的多标签分类**: 实现6维度18类的多标签分类，宏平均F1 > 0.65
3. **智能辨证诊断**: 基于舌象特征提供中医证型分析，专家一致性 > 75%
4. **快速响应**: 端到端诊断时间 < 2秒（P95）
5. **高可用性**: API可用性 ≥ 99%

### 2.2 分阶段目标

#### Phase 1: MVP验证期 (1-2月)
- 舌体分割: mIoU > 0.90
- 主分类准确率: > 75% (健康舌/红舌/白苔/黄苔)
- 云端诊断: 基础Prompt，可用性验证
- 响应时间: < 5s

#### Phase 2: 精度提升期 (3-4月)
- 舌体分割: mIoU > 0.93
- 多标签mAP: > 0.75
- 专家一致性: > 75%
- 响应时间: < 2s (CPU)

#### Phase 3: 工程化期 (5-6月)
- 舌体分割: mIoU > 0.95, 40FPS (CPU)
- 多标签mAP: > 0.82
- 专家一致性: > 80%
- 响应时间: < 1s (本地模式)

## 六、Design Considerations

### 6.1 类别不平衡处理

本数据集存在严重的类别不平衡问题（淡白舌占80.9%），需要在数据层、算法层、评估层三个层面进行应对：

**数据层**:
- 少数类过采样（样本<100的类别）
- 多数类欠采样（样本>1000的类别，控制到80%）
- SMOTE合成中间样本

**算法层**:
- Focal Loss（α=0.25, γ=2）聚焦难样本
- 类别加权Loss（weight = 1/√(count)）
- 分层采样（每batch包含多数类+少数类各50%）
- 困例挖掘（loss Top 10%样本重采样）

**评估层**:
- 使用宏平均F1（而非准确率）
- 分别关注多数类和少数类的指标
- 少数类单独评估

### 6.2 多标签分类设计

原21类标签重构为6维度18类：

| 维度 | 类别数 | 类别详情 |
|------|--------|----------|
| 舌色 | 4 | 淡白/淡红/红/绛紫 |
| 苔色 | 4 | 白苔/黄苔/黑苔/花剥苔 |
| 舌形 | 3 | 正常/胖大/瘦薄 |
| 苔质 | 3 | 薄苔/厚苔/腻苔 |
| 特征标记 | 3 | 红点/裂纹/齿痕 |
| 综合 | 1 | 健康舌（无异常） |

### 6.3 Prompt工程策略

- **结构化输出**: 强制JSON格式，便于解析
- **Few-shot学习**: 提供3-5个典型示例提升一致性
- **分步骤推理**: 要求先给出依据要点，再给出辨证结论
- **置信度输出**: 让模型给出诊断的可信度评分
- **异常检测**: 当特征冲突时要求模型提示

### 6.4 降级与兜底方案

```
API调用 → 成功 → 返回LLM诊断
    │
    └→ 失败(超时/限额) → 本地规则库诊断
                       ├─ 规则匹配：{特征组合} → {预设证型}
                       ├─ 相似度检索：基于历史诊断案例
                       └─ 默认回复：建议线下就医
```

## 七、Technical Considerations

### 7.1 模型选型理由

| 模型 | 选择理由 |
|------|----------|
| BiSeNetV2 + STDCNet2 | 专为分割设计，平衡精度与速度，适合CPU部署 |
| PP-HGNetV2-B4 | 飞桨优化，兼顾精度与速度，ImageNet22k预训练效果好 |
| Focal Loss | 应对类别不平衡，聚焦难样本 |
| Asymmetric Loss | 处理正负样本不平衡 |

### 7.2 数据增强策略

使用Albumentations库进行医学图像专用增强：
- 几何：RandomFlip(0.5)、Rotate(±15°)、Scale(0.8-1.2)
- 颜色：HueSaturationValue(±10%)、RandomBrightness(±20%)
- 医学专用：GaussNoise(5%)、Blur(3%)
- 高级：CutMix(0.2)、MixUp(0.1) - 训练后期启用

### 7.3 部署架构

**本地推理服务**:
- FastAPI服务层
- Paddle Inference引擎（CPU优先，MKL/OpenBLAS）
- Redis缓存（特征结果缓存）
- Celery任务队列（异步调用）

**云端对接**:
- 文心4.5 API（ERNIE-Speed）
- 异步调用（Celery）
- 本地规则库兜底

### 7.4 性能优化

- INT8量化（分割模型）
- FP16量化（分类模型）
- 批处理推理
- OpenVINO/MKL优化
- 结果缓存（Redis）

### 7.5 约束与依赖

**硬件约束**:
- 训练: GPU (V100/A10) 或 云GPU按需
- 部署: CPU优先 (8核+ / 16GB+内存)

**软件依赖**:
- Python 3.9+
- PaddlePaddle 2.6.0
- CUDA 11.8 (训练用)
- FastAPI 0.104.0+

**外部依赖**:
- 文心4.5 API
- Redis
- Celery
