# AI舌诊智能诊断系统 - 临床试验方案

**系统名称**: AI舌诊智能诊断系统
**方案版本**: v2.3
**方案日期**: 2026年2月21日
**方案类型**: 前瞻性、多中心、诊断性临床试验

---

## 一、试验概述

### 1.1 试验背景

AI舌诊智能诊断系统是基于深度学习的舌象辅助诊断软件。为验证其安全性和有效性，依据《医疗器械临床试验管理办法》和《人工智能医疗器械审评要点》，开展本次临床试验。

### 1.2 试验目的

**主要目的**:
1. 验证系统舌体分割的准确性（mIoU ≥ 0.90）
2. 验证系统舌象分类的准确性（多标签mAP ≥ 0.70）

**次要目的**:
1. 评估系统与中医专家诊断的一致性（Kappa ≥ 0.75）
2. 评估系统对不同证型的识别能力
3. 评估系统的临床实用性

### 1.3 试验设计

| 设计要素 | 描述 |
|----------|------|
| **试验类型** | 前瞻性、多中心、诊断性试验 |
| **对照方式** | 金标准对照（中医专家诊断） |
| **盲法设计** | 单盲（专家不知道系统结果） |
| **非劣效界值** | 准确率差异 < 10% |

## 二、试验人群

### 2.1 纳入标准

1. **年龄**: 18-75周岁
2. **性别**: 不限
3. **舌象特征**:
   - 舌体完整、无明显畸形
   - 能够配合拍摄舌部图像
4. **疾病状态**:
   - 健康志愿者
   - 患有各种中医证型患者
5. **知情同意**: 签署知情同意书

### 2.2 排除标准

1. **舌象异常**: 舌体缺损、严重染色、舌苔厚腻无法观察
2. **急症状态**: 需要紧急医疗干预
3. **认知障碍**: 无法理解试验要求
4. **孕期妇女**: 妊娠期或哺乳期妇女
5. **其他**: 研究者认为不适合参加试验

### 2.3 剔除标准

1. 图像质量不符合要求
2. 数据记录不完整
3. 违反试验方案

## 四、试验流程

### 4.1 试验流程图

```
筛选期 → 入组期 → 基线评估 → 图像采集 → 系统诊断 → 专家诊断 → 对比评估 → 数据分析
   ↓        ↓         ↓          ↓          ↓          ↓          ↓          ↓
 知情同意  纳入排除   人口学     标准拍摄    AI系统     3位专家    一致性    统计分析
            检查      信息       条件       分析       独立诊断   评估       报告
```

### 4.2 访视安排

| 访视 | 时间 | 评估内容 |
|------|------|----------|
| **筛选访视** | Day -7 to 0 | 知情同意、纳排检查 |
| **基线访视** | Day 0 | 人口学信息、病史采集 |
| **试验访视** | Day 0 | 舌象采集、系统诊断、专家诊断 |
| **结束访视** | Day 1 | 数据完整性检查 |

### 4.3 舌象采集标准

**设备要求**:
- 相机: 1200万像素以上
- 光源: 标准LED光源，色温5000-5500K
- 背景: 白色或灰色背景布

**拍摄要求**:
1. 受试者自然伸舌，不过分用力
2. 舌体放松，充分暴露
3. 拍摄距离: 15-20cm
4. 拍摄角度: 正面，与舌面垂直
5. 图像格式: JPEG/PNG，分辨率≥512×512

**质量控制**:
- 图像清晰度评分 ≥ 3分（5分制）
- 舌体面积占比 ≥ 60%
- 无明显反光和阴影

## 五、评价指标

### 5.1 主要评价指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| **分割mIoU** | 系统分割与金标准分割的平均IoU | ≥0.90 |
| **分类mAP** | 多标签分类的平均精度均值 | ≥0.70 |

### 5.2 次要评价指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| **专家一致性Kappa** | 系统诊断与专家诊断的一致性 | ≥0.75 |
| **准确率** | 正确分类的比例 | ≥85% |
| **敏感性** | 真阳性率 | ≥80% |
| **特异性** | 真阴性率 | ≥90% |
| **F1分数** | 准确率和召回率的调和平均 | ≥0.75 |

### 5.3 安全性评价指标

| 指标 | 说明 |
|------|------|
| **不良事件** | 试验过程中发生的任何不良事件 |
| **严重不良事件** | 导致住院、残疾或死亡的事件 |
| **设备相关故障** | 系统运行中的技术故障 |

## 六、统计分析

### 6.1 分析数据集

| 数据集 | 定义 | 用途 |
|--------|------|------|
| **FAS集** | 全全分析集（Full Analysis Set） | 主要分析 |
| **PPS集** | 符合方案集（Per Protocol Set） | 敏感性分析 |
| **SS集** | 安全集（Safety Set） | 安全性分析 |

### 6.2 统计方法

#### 6.2.1 主要终点分析

**分割mIoU**:
- 计算方法: IoU = 交集/并集
- 统计方法: 单样本t检验（与目标值0.90比较）
- 显著性水平: α = 0.05（单侧）

**分类mAP**:
- 计算方法: 各类别AP的均值
- 统计方法: 单样本t检验（与目标值0.70比较）
- 显著性水平: α = 0.05（单侧）

#### 6.2.2 次要终点分析

**一致性分析**:
- Kappa系数: 衡量系统与专家的一致性
- 95%置信区间: Bootstrap法
- 判断标准: Kappa ≥ 0.75为良好一致性

**准确率分析**:
- 点估计: 正确分类比例
- 95%置信区间: Wilson score interval
- 组间比较: 卡方检验或Fisher精确检验

### 6.3 缺失数据处理

| 缺失类型 | 处理方法 |
|----------|----------|
| **图像质量不达标** | 剔除不计入分析 |
| **专家诊断缺失** | 不纳入一致性分析 |
| **系统诊断失败** | 记录为失败，纳入安全性分析 |

## 七、质量控制

### 7.1 图像质量控制

**质控流程**:
1. 拍摄时实时质量评估
2. 上传前自动质量检查
3. 专家人工审核
4. 不合格图像重新采集

**质量标准**:
- 清晰度: ≥ 3分（5分制）
- 完整性: 舌体完整暴露
- 一致性: 与标准拍摄条件一致

### 7.2 数据管理

**数据录入**:
- 电子病例报告表(eCRF)
- 双人双录核对
- 逻辑核查

**数据存储**:
- 加密存储
- 定期备份
- 访问控制

**数据监查**:
- 独立数据监查委员会(IDMC)
- 定期安全性报告
| 监查内容 | 监查频率 |
|----------|----------|
| 数据完整性 | 每月 |
| 方案依从性 | 每月 |
| 安全性事件 | 实时 |

## 八、伦理学考虑

### 8.1 知情同意

**知情同意书内容**:
1. 试验目的和流程
2. 预期受益和风险
3. 自愿参与原则
4. 随时退出权利
5. 隐私保护措施

**知情同意流程**:
1. 研究者解释试验内容
2. 受试者提问和解答
3. 充分考虑时间（≥24小时）
4. 签署知情同意书

### 8.2 隐私保护

**保护措施**:
- 图像去标识化（EXIF清理、文件名哈希）
- 数据加密存储和传输
- 限制访问权限
- 数据留存180天后自动删除

### 8.3 风险受益评估

**潜在风险**:
- 舌部图像采集: 无创、无风险
- 误诊风险: 低（仅辅助诊断）

**潜在受益**:
- 诊断参考: 为中医师提供辅助
- 健康评估: 了解健康状况

**风险受益比**: 正向

## 十一、安全性管理

### 11.1 不良事件定义

**不良事件(AE)**: 受试者接受试验后出现的任何不利医学事件

**严重不良事件(SAE)**:
- 导致死亡
- 危及生命
- 需要住院或延长住院
- 导致永久或显著残疾
| 事件级别 | 报告时限 |
|----------|----------|
| SAE | 24小时内 |
| 其他AE | 每月汇总 |

### 11.2 安全性监查

**监查委员会**:
- 独立数据监查委员会(IDMC)
- 定期审查安全性数据

|----------|------|
| SAE汇总 | 即时 |
| AE趋势 | 每季度 |
